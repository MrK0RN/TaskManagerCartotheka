# Развёртывание на webhosting с PostgreSQL

Версия сайта для хостинга с PostgreSQL (без Docker). Код идентичен основной версии.

## Требования

- **PHP 8.2** с расширениями `pdo`, `pdo_pgsql`
- **Apache** (или аналог) с document root, указывающим на корень этой папки (или поддиректорию — тогда настройте `BASE_URL` в `config/config.php`)
- **PostgreSQL 16** (или совместимая версия) — БД может быть на том же сервере или удалённо

## Шаг 1. База данных

1. Создайте базу и пользователя в PostgreSQL.
2. Выполните схему (из каталога с файлами или укажите полный путь к `sql/schema.pgsql`):

   ```bash
   psql -U ваш_пользователь -d ваша_бд -f sql/schema.pgsql
   ```

3. При необходимости выполните миграции:
   - `sql/telegram_subscribers_migration.pgsql` — подписчики бота
   - `sql/tasks_migration.pgsql` — только если таблицу `tasks` не создаёт приложение (таблицы `portrait_meetings` и `tasks` создаются автоматически при первом подключении приложения, если их ещё нет; см. `config/database.php`)

## Шаг 2. Конфигурация

1. Скопируйте файл конфигурации и заполните данные БД:

   ```bash
   cp config/local.example.php config/local.php
   ```

2. Откройте `config/local.php` и укажите хост, порт, имя БД, пользователя и пароль PostgreSQL.
3. Если сайт открывается из подпапки (например `https://example.com/app/`), измените `BASE_URL` в `config/config.php` (например `define('BASE_URL', '/app/');`).

## Шаг 3. Загрузка файлов

Загрузите всё содержимое этой папки в корень сайта на хостинге (или в нужную подпапку). При обновлении не перезаписывайте `config/local.php` — в нём хранятся ваши секреты.

## Безопасность

- Закройте доступ по HTTP к каталогам `config/` и `sql/` (настройки веб-сервера или `.htaccess`), чтобы к ним нельзя было обратиться из браузера.
- На продакшене отключите вывод ошибок в `config/config.php`: `ini_set('display_errors', 0);` и настройте логирование.

## Telegram-бот

Бот подключается к той же PostgreSQL. Его можно запускать:

- на том же сервере (cron или фоновый процесс);
- на отдельном VPS.

Подробности: [telegram_bot/README.md](telegram_bot/README.md). Задайте переменные окружения `DB_HOST`, `DB_PORT`, `DB_NAME`, `DB_USER`, `DB_PASS` и `SITE_URL` (публичный URL сайта) и при необходимости `TELEGRAM_BOT_TOKEN`.

## Голосовой ввод (Yandex SpeechKit)

По желанию в `config/local.php` можно задать:

```php
define('YANDEX_SPEECHKIT_API_KEY', 'ваш_ключ');
define('YANDEX_STT_LANG', 'ru-RU');
```

Без этих настроек голосовой ввод в форме портрета работать не будет (остальной функционал — будет).
